# T512-OPSEC

La plupart des démarches OPSEC traitent matériel, logiciel et firmware comme trois problèmes séparés. Les points de rupture apparaissent précisément à leurs interfaces. Ce projet les traite simultanément.

Le ThinkPad L512 n'est pas conçu comme un objet furtif. Un profil rare attire. Un fantôme finit par émerger. La posture repose sur la banalité statistique, pas sur la furtivité. Le laptop est un terminal non fiable opérant dans un espace symbolique contrôlé. Il ne négocie rien, ne voit jamais le réseau radio, ne constitue pas un point de décision. Toute exposition est déplacée, filtrée, médiée par des couches externes.

Ce dépôt expose les hypothèses, les décisions, les erreurs et les limites. Il documente ce qui a été abandonné autant que ce qui a été construit. Il assume que le lecteur sait ce qu'est me_cleaner.
